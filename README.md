# рҹ“қ Notes вҖ” Flutter + Cloud Firestore

A simple, full-stack **Notes Management** mobile app built with **Flutter** and backed by **Cloud Firestore**. Create, edit, delete, and browse your notes in real time вҖ” changes sync to the cloud the instant you tap save.

The app follows a clean **MVVM + Repository + DataSource** architecture so the UI stays free of business logic.

---

## вңЁ Features

- вңҚпёҸ **Create** notes with a title and description
- вңҸпёҸ **Edit** existing notes inline
- рҹ—‘пёҸ **Delete** notes with a confirmation dialog
- вҳҒпёҸ **Real-time sync** вҖ” every device viewing the list sees changes the moment they happen
- рҹҢ— **Light & dark mode** that follows the OS theme (Material 3)
- рҹӣЎпёҸ **Typed errors** surfaced to the UI (e.g., explicit "permission denied" messages)
- рҹ§ұ **Clean separation** вҖ” views import only a single `NotesController`; nothing else

## рҹҸ—пёҸ Architecture

The codebase is split into four layers. The dependency arrow only points **inward**:

```
views/      в”Җв–¶  controllers/  в”Җв–¶  repositories/  в”Җв–¶  services/  в”Җв–¶  cloud_firestore
(UI)             (state + actions)   (orchestration)    (data source)    (raw SDK)
   в–І                   в”Ӯ
   в””в”Җв”Җ Provider в—Җв”Җв”Җв”Җв”Җв”Җв”Җв”ҳ   (ChangeNotifier exposed to the widget tree)
```

| Layer | Folder | Responsibility |
|---|---|---|
| **View** | `lib/views/` | Pure UI вҖ” widgets, layouts, navigation. No Firestore imports. |
| **Controller** | `lib/controllers/` | `NotesController` вҖ” `ChangeNotifier` exposing `notes`, `isLoading`, `error`, plus action methods (`addNote`, `updateNote`, `deleteNote`). Owns the stream subscription and typed `NotesError` hierarchy. |
| **Repository** | `lib/repositories/` | Thin orchestration layer between the controller and the data source. No mapping, no business rules. |
| **Service / Data source** | `lib/services/` | The only place that talks to `cloud_firestore`. Wraps `FirebaseFirestore` and exposes typed `Stream<List<Note>>` + `Future<void>` operations. |

**Other folders:**

- `lib/models/` вҖ” immutable `Note` data class with `Note.fromFirestore(...)` and `toMap()`.
- `lib/core/` вҖ” cross-cutting concerns: `constants/`, `theme/`, `routes/`, `widgets/`, `utils/`.
- `lib/firebase_options.dart` вҖ” generated by FlutterFire CLI for the Firebase project.

## рҹҡҖ Getting Started

### Prerequisites

- Flutter SDK `^3.12.2` ([install guide](https://docs.flutter.dev/get-started/install))
- A Firebase project (the included `firebase_options.dart` is wired to `note-application-crud`)

### 1. Clone & install dependencies

```bash
git clone <your-repo-url>
cd task_app
flutter pub get
```

### 2. Configure Firebase (one-time)

If you're using the included Firebase project, skip to step 3. Otherwise:

```bash
# Install the FlutterFire CLI
dart pub global activate flutterfire_cli

# Re-generate firebase_options.dart for your own project
flutterfire configure
```

### 3. Deploy the Firestore security rules

This repo includes a development-grade `firestore.rules` file. Deploy it so writes are allowed:

```bash
# Install the Firebase CLI (if you don't have it)
npm install -g firebase-tools        # or download firebase.exe from firebase.tools

# Sign in and deploy
firebase login
firebase deploy --only firestore:rules
```

> **вҡ пёҸ Dev rules only:** the shipped rules allow open read/write to the `notes` collection. Tighten them (e.g., `request.auth != null`) before going to production.

### 4. Run the app

```bash
flutter run                       # auto-detects a connected device
flutter run -d chrome             # web
flutter run -d <device-id>        # specific device
```

## рҹ”Ң Firestore Schema

Single collection: **`notes`**

| Field         | Type                  | Notes                                  |
|---------------|-----------------------|----------------------------------------|
| `title`       | `string`              | Required, max 100 chars (client-side)  |
| `description` | `string`              | Required, max 1000 chars (client-side) |
| `createdAt`   | `timestamp` (server)  | Set via `FieldValue.serverTimestamp()` |

Document IDs are auto-generated by Firestore.
